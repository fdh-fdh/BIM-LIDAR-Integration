# Point Cloud Outlier Cleaning and Element Clustering

Welcome to the Point Cloud Outlier Cleaning and Element Clustering repository! 
This repo provides tools and scripts for processing point cloud data which is detect as conflict with the BIM Model, 
with a focus on outlier detection and element clustering, for preparing feature extractions in order to update the BIM model

## Introduction
The input point cloud represents a subset of LIDAR scans, specifically the Conflict Point Cloud. 
This subset is generated by comparing occupancy information with Building Information Modeling (BIM) data.Algorithmn: open3d DBSCAN

## Features

- **Outlier remover:** Detect and remove outliers from point cloud data using methods such as statistical analysis, density-based clustering, and more.

- **Clustering:** Group point cloud elements based on proximity, density, or other criteria to identify distinct objects or structures.



Parameter explaination : 
Epsilon (eps):
eps is a distance threshold that defines the maximum distance between two points for one point to be considered as part of the neighborhood of the other.
It essentially defines the radius around each data point within which we look for other data points to form a cluster.
Points within an eps distance from each other are considered neighbors and are grouped together in the same cluster.
Minimum Points (min_samples or min_pts):
min_samples (or min_pts) is the minimum number of data points that must be within the eps radius of a point for it to be considered a core point.
Core points are central to a cluster and are used to initiate the formation of clusters. They must have at least min_samples other data points in their eps neighborhood to be considered a core point.
Non-core points that are within the eps neighborhood of a core point, but do not have enough neighbors to be considered core themselves, are considered border points.
Points that are not core points and do not belong to any cluster are considered noise points.

Fine tuning parameter
A core point with at least min_samples neighbors within an eps radius forms the core of a cluster.
All the points within the eps neighborhood of a core point, including the core point itself, belong to the same cluster.
If a non-core point is within the eps neighborhood of a core point, it becomes part of the same cluster.
Data points that are not core points, and are not within the eps neighborhood of any core point, are treated as noise and do not belong to any cluster.
Choosing appropriate values for eps and min_samples is crucial for achieving meaningful clustering results:

A smaller eps value will result in tighter clusters but may lead to more noise points.
A larger eps value will result in larger and more inclusive clusters but may also merge separate clusters into one.
Adjusting min_samples determines how strict the algorithm is in recognizing core points. A larger value requires more points to be densely packed to form a cluster.
The choice of eps and min_samples depends on the characteristics of your data and the goals of your clustering analysis. Experimenting with different parameter values and visualizing the results can help you determine the best settings for your specific dataset.
